================================================================================
AGENT RULES
================================================================================

  - Read ONLY llm.txt for repo/architecture questions.
  - Do NOT glob/grep the entire repo unless llm.txt is insufficient.
  - For deltas, read RECENT FILE CHANGES then open only those paths.
  - Use repo_map --depth 3 only when the user explicitly requests a full tree scan.
  - Refresh changelog after pulls: arka repo index  (or arka llm sync)
  - Skill: repo_context — returns relevant llm.txt sections (symbolic barrier default)
  - Skill: repo_map — live tree scan escape hatch for explicit "map this repo" requests

================================================================================
PROJECT SUMMARY
================================================================================

Arka is a cross-platform terminal AI agent (macOS, Linux, Windows). Users type
plain English; Arka routes to 70+ local skills with deterministic offline rules
first, then LLM fallback. Skills run locally; LLM calls failover across 24+
providers (Gemini, Groq, Ollama, etc.). Security gates (prompt injection, risky
action confirmation) are on by default. Package name on PyPI: `arka-agent`.

Entry point: `arka` CLI → `src/arka/cli.py:main`
Core flow: NL text → router → skill command line → dispatch → bin shim → package module

================================================================================
ARCHITECTURE
================================================================================

ROUTING PIPELINE (see docs/concepts/routing.mdx)
------------------------------------------------
1. Direct skill name     — first token in _agent_all_skills → zero tokens
2. Third-party plugins   — ~/.config/arka/skills/*/skill.json triggers
3. Fugu rules            — integrations/fugu.py (optional fast path)
4. Fish symbolic rules   — config.fish _agent_offline_route_cmd / _agent_guess_route
5. Python symbolic extras — routing/symbolic.py route_offline_extras (no fish needed)
   repo_context skill — llm.txt lookup (default for repo/architecture NL)
   repo_map skill — full tree scan (explicit map requests only)
6. LLM route             — compact skill catalog, task=route (skill_profiles)
7. Correction layer      — fixes weak LLM picks without second LLM call
8. Dispatch              — dispatch.py runs the chosen skill

Route modes (ROUTE_MODE env, router.py _route_mode):
  symbolic (default) | ai | symbolic_only | ai_only

Operation modes (ARKA_MODE env or ~/.config/arka/mode, core/mode.py):
  agent (default) | ask | plan | debug | multitask
  - ask: read-only skills OK; risky actions blocked → web_answer fallback
  - plan: structured steps printed, no skill execution
  - debug: LLM_VERBOSE + route preview details
  - multitask: spawns background sub-agents (integrations/subagent.py)
  CLI: arka mode [ask|plan|agent|debug|multitask|list]

Python router: src/arka/router.py
  - route(text) → Route(skill, source, kind)
  - Uses fish subprocess when fish + config.fish available
  - Falls back to routing/symbolic.py when fish absent or symbolic_only

Fish router: src/arka/fish/config.fish (~10k lines)
  - Canonical skill registry: _agent_all_skills
  - NL→skill: _agent_offline_route_cmd, _agent_guess_route
  - Skill execution: _agent_dispatch → _agent_dispatch_one
  - Synced to bundled/config.fish for pip installs

DISPATCH
--------
src/arka/dispatch.py
  - run_skill(skill_line) — inline handlers for common skills, else run_script/run_fish_skill
  - run_script("arka_*.py", args) — resolves script via paths.entry_script
  - apply_env() sets INSTALL_HOME, CONFIG_DIR, CACHE_DIR

BIN SHIMS → PACKAGE MODULES
---------------------------
bin/arka_*.py are thin shims → arka._bootstrap.run_legacy_module(qualname)
Legacy name map: src/arka/_bootstrap.py LEGACY_MODULES (118 entries)
Shim path helper: bin/_shim_path.py (adds src/ to sys.path for dev)

BUNDLED SYNC (pip wheel runtime)
--------------------------------
scripts/sync_bundled.py copies:
  - fish/config.fish → bundled/config.fish
  - bin/*.py → bundled/
  - requirements, env.example, aie/, privategpt/, whatsapp/
Run after adding/changing bin shims. Test: tests/test_bundled_sync.py
  asserts every bin/arka_*.py exists in src/arka/bundled/

PATHS
-----
src/arka/paths.py
  - arka_home() — bundled/ when installed; repo root in editable dev if unsynced
  - config_dir() — ~/.config/arka (or legacy ~/.config/fish)
  - cache_dir() — platformdirs user cache
  - entry_script(name) — bin/ in dev, flat in bundled/
  - fish_config() — bundled/config.fish or user override

LLM LAYER
---------
src/arka/llm/fallback.py — LLMFallback.complete(), provider chain, key rotation
src/arka/llm/providers.py — provider specs, base URLs, model IDs
src/arka/llm/servers.py — local vLLM/Ollama reachability
src/arka/llm/skill_profiles.py — TASK_PROFILES + SKILL_TASK_MAP (per-skill model task)
src/arka/llm/api_keys.py — multi-key rotation per provider
User overrides: ~/.config/arka/llm-skill-models.json

Default chain: Gemini → Groq → Ollama (see DEFAULT_CHAIN in fallback.py)

VISION STACK
------------
src/arka/vision/describe.py — images: local vLLM, Ollama vision, Gemini; OCR grounding
src/arka/vision/screen.py — countdown + capture + describe_screen
src/arka/vision/video.py — ffmpeg frame sampling + people/location in video
Each has nl_to_argv() for symbolic routing; bin shims: arka_describe_image.py,
arka_describe_video.py (screen uses fish + describe.py)

MCP
---
Client: integrations/mcp_client.py, mcp_manager.py, mcp_cli.py — ~/.config/arka/mcp.json
Server: integrations/mcp_server.py — Arka as stdio MCP (arka_ask, arka_remember, arka_skill, …)
CLI: arka mcp … | bin/arka_mcp.py, bin/arka_mcp_server.py

THIRD-PARTY SKILLS
------------------
src/arka/agent/skills.py — discover/install/run plugins (skill.json)
Search paths: SKILLS_PATH, ~/.config/arka/skills, package skills/, bundled skills/
Examples: src/arka/skills/examples/demo_echo/, skills/life_sciences/
Fish integration: _arka_third_party_skill_names, _arka_run_third_party_skills

KEY FILES FOR COMMON TASKS
================================================================================

ADD A NEW SKILL
  1. Implement module under src/arka/<area>/ with main() or nl_to_argv()
  2. Add bin/arka_<name>.py shim (copy pattern from bin/arka_flow.py)
  3. Run: python scripts/sync_bundled.py
  4. Register in fish/config.fish:
     - Add to _agent_all_skills
     - Add _agent_is_* / _agent_route_* or inline rule in _agent_offline_route_cmd
     - Add case in skill help / dispatch function if needed
  5. Add route_* in routing/symbolic.py + route_offline_extras tuple
  6. Map in llm/skill_profiles.py SKILL_TASK_MAP if LLM-backed
  7. Add LEGACY_MODULES entry in _bootstrap.py if legacy flat import needed
  8. Test: tests/test_<skill>.py + phrase in tests/test_nl_routing_coverage.py

FIX ROUTING / NL MATCHING
  - Fish (primary): src/arka/fish/config.fish (_agent_offline_route_cmd, _agent_guess_route)
  - Python mirror: src/arka/routing/symbolic.py (route_offline_extras order matters)
  - Router orchestration: src/arka/router.py
  - Preview: arka route "your phrase" or ROUTE_MODE=symbolic_only pytest
  - Learned routes: agent/route_learn.py, routing/learned.py

LLM PROVIDER / FAILOVER
  - src/arka/llm/fallback.py, providers.py, api_keys.py, servers.py
  - Per-skill models: llm/skill_profiles.py + ~/.config/arka/llm-skill-models.json
  - CLI: arka ai-skill-model, arka ai-models

VISION (describe_image / describe_screen / describe_video)
  - src/arka/vision/describe.py (core backends, prompts, resize)
  - src/arka/vision/screen.py, video.py (NL parse + capture/sampling)
  - Tests: tests/test_describe_*.py

MCP

    Persists ~/.config/arka/code-project.json; gates agent_code, write_script, self_improve; goal scopes when active
    agent_code without init hints: arka code init . (cwd shown)
  - arka self improve [target] [--apply] — self-improvement (src/arka/agent/self_improve.py)
    Default plan-only: analyze llm.txt + git log + pytest → LLM/heuristic plan; --apply runs goal agent
    Memory: ~/.config/arka/self-improve-memory.json; blocks .env/secrets/node_modules + destructive git
    NL: improve arka | self improve routing | improve arka llm fallback | loop self "fix tests"
    --apply requires agent mode; SELF_IMPROVE_MAX_STEPS / _MAX_ROUNDS env limits
    Dedicated skill: self-improve-routing (Codex skill for routing-focused self-improvement requests)
  - arka setup runs arka repo index when llm.txt changelog is stale

================================================================================
CONVENTIONS & GOTCHAS
================================================================================

FISH vs PYTHON ROUTING
  - Full 120+ symbolic rules live in fish/config.fish (~10k lines)
  - Python symbolic.py mirrors many routes for fish-less / symbolic_only mode
  - When adding NL patterns: update BOTH unless fish-only (platform-specific shell)
  - route_offline_extras() order is priority — first match wins

BUNDLED COPIES
  - Never edit src/arka/bundled/ alone — regenerated by sync_bundled.py
  - CI test fails if bin/arka_*.py missing from bundled/
  - fish/config.fish source → bundled/config.fish via sync

BIN SHIM PATTERN
  bin/arka_foo.py:
    import _shim_path; _shim_path.ensure()
    from arka._bootstrap import run_legacy_module
    run_legacy_module("arka.module.path")

SKILL_PROFILES
  - LLM task profiles: route, chat, summarize, research, agent, pdf, flow, …
  - SKILL_TASK_MAP maps skill name → profile for model selection

NL_TO_ARGV PATTERN
  Skills that accept natural language expose nl_to_argv(text) → list[str]
  route_* in symbolic.py calls nl_to_argv and builds "skill arg1 arg2"

SECURITY
  - core/security.py, fish __agent_classify, _arka_confirm_risky_action
  - Blocks destructive shell patterns; confirms install/delete/send

ENV / CONFIG
  - User .env: ~/.config/arka/.env (template: .env.example, src/arka/env.example)
  - ROUTE_MODE, INSTALL_HOME, CONFIG_DIR, CACHE_DIR, SKILLS_PATH
  - Legacy path: ~/.config/fish/.env still honored

THIRD-PARTY skill.json
  - name, type (python|fish), entry, triggers[], requires{}, metadata{}
  - Install dir: ~/.config/arka/skills/<name>/

PACKAGE DATA (pyproject.toml tool.setuptools.package-data)
  - bundled/*, fish/*, skills/**/*, personas templates, requirements/*.txt
